## 가치관

{==
숲을 볼 때와 나무를 볼 때를 구분하려 노력하는 주니어 엔지니어입니다.
==}

기술의 대중화를 통해 좋은 사회적 가치를 전달하고 싶어 프로그래밍 공부를 처음 시작했습니다. 구체적으로 일상 속에서 불편하고 당연하게 여기는 것들을 기술로 해결했습니다. 예를 들어 Slack 채널 내에서 스레드를 통해 수동으로 확인하던 기상 스터디의 불편함을 Slack API, AWS Lambda와 CloudWatch Events를 통해 자동으로 확인하고 결과를 알려주는 [웹훅 시스템](#we-ake-up)을 구축하여 해결하였습니다. 또한 프로젝트를 비대면으로 진행할 때 지난 회의록을 Notion을 통해 여러 페이지 깊이(Depth)를 겪으며 확인하는 것보다 사용량이 가장 높고 많은 대화가 오가는 Slack에서 확인하는 게 능률적일 것이라 생각되어 공식적인 Notion API가 존재하지 않던 당시에 네트워크를 통해 통신을 확인하여 직접 [회의록 확인 백오피스 웹훅 시스템](#notion-slack-통합)을 구축하였습니다.

추가적으로 새로운 기술을 사용하는 데 있어 두려움이 없고 오픈 소스에 관심이 많습니다. 최근 [머신러닝을 활용한 공부 집중도 분석 및 학습 리포트 제공 웹 서비스](#studeep)를 구축한 적 있습니다. 해당 서비스를 구축할 때 클라이언트에서 Tensorflow.js를 활용하여 처리된 머신러닝 데이터를 서버로 전달하면 본인의 집중 상태를 동일 공부방 내의 다른 사용자에게 실시간으로 표시하게 했어야 했는데 처음 경험해보는 FastAPI, SocketIO를 성공적으로 적용하였습니다. 

끝으로 이런 다양한 경험을 통해 좋은 사회적 가치를 전달할 수 있는, 좋은 기술의 대중화는 단지 기술로만 완성되지 않는다는 걸 알게 되었습니다. 좋은 코드를 작성하는 것만큼 소통과 문서화, 그리고 더 나아가 비즈니스 지식까지. 기술이라는 나무만큼 협업과 비즈니스라는 숲을 보는 것도 중요하다는 것을 알게 되었습니다. 그래서 프로젝트에서 [프론트엔드의 CI/CD](https://www.weekwith.me/ko/devlog/next-js/ci-cd-static-webstie-by-github-actions-and-aws-s3/) 또한 직접 AWS S3와 CloudFront 그리고 GitHub Actions를 통해 구현해보고 또 비개발 직군과의 원활한 소통을 위해 현재 [소프트-웨어](https://maily.so/weekwith.me)라는 메일링 서비스도 운영하며 약 180명 정도의 구독자를 보유하는 등 나무뿐 아니라 숲을 보기 위한 노력도 끊임없이 하는 중입니다.
